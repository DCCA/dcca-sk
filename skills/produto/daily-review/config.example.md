# Config - daily-review

Copie este arquivo para **`config.md`** (no mesmo diretório) e preencha com os seus dados. O `config.md` é ignorado pelo git - seus dados reais (nomes, canais, pessoas) nunca vão para o repositório, então a skill continua agnóstica a empresa.

Campo vazio: a skill segue sem ele, com menos precisão.

- **Fontes ativas:** [calendário, e-mail, Slack, GitHub, Jira, transcrições - só as que sua empresa usa]
  A skill consulta apenas estas; adapte à sua stack (Jira ou Linear; Teams no lugar de Slack).
- **Handles no Slack:** [@seu.handle, apelidos]
  Para detectar menções diretas.
- **Canais prioritários (allowlist):** [#squad, ...]
  Lidos por completo; fora da lista, só DMs, menções e threads onde você respondeu.
- **VIPs no e-mail:** [chefe, pares, clientes]
  Prioridade mesmo sem estrela.
- **Glossário:** [pessoas, produtos, siglas]
  Para normalizar termos que a transcrição distorce.
